### 2. 测试最佳实践

#### 2.1 使用 SQLite 内存模式

**优点**:
- 测试速度快（无需启动 PostgreSQL）
- 测试隔离（每个测试独立数据库）
- 支持自动迁移（AutoMigrate）

**示例**:
```go
db, err := gorm.Open(sqlite.Open(":memory:"), &gorm.Config{})
db.AutoMigrate(&domain.User{}, &domain.Post{})
```

---

#### 2.2 使用 testify/assert

**优点**:
- 错误信息清晰
- 支持多种断言（Equal, Error, NoError, True, False）
- 可读性强

**示例**:
```go
assert.NoError(t, err)
assert.Equal(t, "testuser", user.Username)
assert.NotZero(t, user.ID)
```

---

#### 2.3 使用 Gin 测试工具

**优点**:
- 可以测试 HTTP 处理器而不启动服务器
- 支持设置请求头（Authorization, Content-Type）
- 响应可以解析为 map 进行验证

**示例**:
```go
req, _ := http.NewRequest("POST", "/api/v1/posts", bytes.NewBuffer(jsonBody))
req.Header.Set("Authorization", "Bearer "+token)
w := httptest.NewRecorder()
r.ServeHTTP(w, req)
assert.Equal(t, http.StatusCreated, w.Code)
```

---

#### 2.4 子测试（t.Run）

**优点**:
- 测试组织清晰
- 可以单独运行某个子测试
- 测试报告易读

**示例**:
```go
t.Run("CreatePost", func(t *testing.T) {
    // 测试创建文章
})

t.Run("UpdatePost", func(t *testing.T) {
    // 测试更新文章
})
```

## 📝 关键学习点

### 1. 为什么需要集成测试？

**问题**: 单元测试都通过了，为什么还需要集成测试？

**答案**: 
- 单元测试只能验证单个模块的功能
- 集成测试能发现模块间集成问题（如数据库事务、API 调用）
- 完整用户流程测试能验证系统整体可用性

**实践经验**:
- 单元测试：快速反馈（秒级）
- 集成测试：慢一些（分钟级），但更准确
- 生产环境：两者都需要

---

### 2. 如何计算测试覆盖率？

**命令**:
```bash
cd examples/reference-blog
go test ./... -coverprofile=coverage.out
go tool cover -html=coverage.out -o coverage.html
```

**查看报告**:
- 打开 `coverage.html`
- 绿色：已覆盖
- 红色：未覆盖
- 可以点击文件名查看详细覆盖情况

---

### 3. 测试覆盖率 > 80% 够吗？

**观点**: 80% 是一个合理的目标，但不是终点

**原因**:
- 100% 覆盖率不代表没有 bug（可能漏掉边界情况）
- 80% 覆盖率可以保证大部分代码被测试过
- 重要的是测试质量，而不是数量

**建议**:
- 核心业务逻辑：100% 覆盖
- 辅助函数（如工具函数）：可以略低
- 错误处理路径：一定要覆盖

## 🤔 遇到的问题和解决方案

### 问题 1: 如何模拟认证中间件？

**解决方案**:
- 在测试中，先调用 `authService.Login()` 获取 token
- 在 HTTP 请求中设置 `Authorization: Bearer <token>` 请求头
- 不需要真正启动认证中间件，直接测试 handler

**示例**:
```go
token, _, _ := authService.Login("testuser", "password123")
req.Header.Set("Authorization", "Bearer "+token)
```

---

### 问题 2: 如何测试数据库事务？

**解决方案**:
- 使用 SQLite 内存模式（每个测试独立数据库）
- 在 `setupTestDB()` 中创建数据库和迁移
- 每个测试都是独立的，不会相互影响

**示例**:
```go
func setupTestDB(t *testing.T) *gorm.DB {
    db, err := gorm.Open(sqlite.Open(":memory:"), &gorm.Config{})
    db.AutoMigrate(&domain.User{}, &domain.Post{})
    return db
}
```

---

### 问题 3: 如何测试并发？

**解决方案**:
- 使用 Go 的 goroutine 和 channel
- 在测试中启动多个 goroutine 并发执行
- 使用 `sync.WaitGroup` 等待所有 goroutine 完成

**示例**:
```go
func TestConcurrentRequests(t *testing.T) {
    done := make(chan bool)
    for i := 0; i < 10; i++ {
        go func() {
            // 并发执行
            done <- true
        }()
    }
    for i := 0; i < 10; i++ {
        <-done
    }
}
```
